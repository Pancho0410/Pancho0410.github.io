Algunos Proyectos realizados

Estos son solo algunos de los proyectos elaborados durante la carrera, los cuales demuestran capacidad técnica, autonomía y experiencia práctica en la implementación de soluciones tecnológicas.

StatPy Convertor – Proyecto Académico (OLC1)

Solución de software enfocada en aplicar las fases de análisis léxico y sintáctico de un compilador, con el objetivo de ofrecer dos funcionalidades principales: traducción automática del lenguaje StatPy a Python y generación de reportes estadísticos a partir de archivos JSON.

Lenguaje de desarrollo: Java
Tecnologías aplicadas: JFlex · CUP · Parsing manual de JSON · Reportes HTML · JFreeChart (para gráficas)

El sistema analiza código StatPy, construye tablas de símbolos, reconoce estructuras de control (if, switch, for, while, funciones, variables) y genera traducción a Python, así como reportes estadísticos visuales (barras y pie). Incluye editor de texto, consola, analizador JSON integrado y manejo de errores léxicos/sintácticos.

QueryCrypter – Proyecto Académico (OLC1)

Interpretador basado en SQL desarrollado de forma individual para el curso de Organización de Lenguajes y Compiladores 1. Reconoce instrucciones DDL y DML (creación, modificación y consulta de datos en memoria), siguiendo principios de análisis léxico y sintáctico.

Lenguaje de desarrollo: JavaScript

Tecnologías utilizadas: Jison (analizador léxico/sintáctico), Node.js, HTML para reportes, Graphviz para AST.

El sistema permite interpretar código propio similar a SQL, generar tabla de símbolos, errores, AST y resultados en consola. Además, permite crear, modificar y consultar tablas en memoria.

XSQL – Proyecto Académico (OLC2)

Sistema administrador de bases de datos desarrollado en pareja para el curso de Organización de Lenguajes y Compiladores 2. Implementa un IDE propio con editor de código, ejecución de consultas, navegación de bases de datos y generación automática de reportes.

Lenguaje de desarrollo: Python

Tecnologías aplicadas: PLY/SLY (análisis léxico y sintáctico) · XML para almacenamiento de datos · Graphviz para AST · Reportes HTML · Sistema de archivos como motor de BD.

Patrón de diseño implementado: Intérprete (Interpreter Pattern) aplicado para procesar y ejecutar instrucciones XSQL, permitiendo la traducción, análisis y ejecución de comandos mediante gramáticas dirigidas por la sintaxis.

El lenguaje XSQL soporta instrucciones DDL, DML y SSL para manipulación de datos, creación de tablas, funciones, procedimientos y consultas. El proyecto demuestra el diseño de un motor de base de datos completo a nivel académico.

Sistema Inteligente de Seguridad y Gestión de Tráfico Urbano – ARQUI2

Proyecto grupal de Arquitectura de Computadoras y Ensambladores 2 que integra IoT e Inteligencia Artificial para monitorear tráfico y seguridad en una maqueta del Centro Histórico (Zona 1, Guatemala). El sistema combina sensores (tráfico, humo, sismo, botones de pánico), microcontroladores Arduino, una Raspberry Pi como nodo central, comunicación vía MQTT, API REST, base de datos y visualización para la ciudadanía mediante una aplicación móvil.


Rol personal: Participación en las tres fases del proyecto (prototipo físico, integración IoT y módulo de IA), con énfasis en el desarrollo de la aplicación móvil en Flutter, encargado de consumir la API, mostrar el estado del tráfico, alertas de seguridad, rutas de buses simulados y paneles de información para el usuario.


Tecnologías clave: Flutter · Dart · Arduino · Raspberry Pi · MQTT · API REST (backend en Raspberry) · Base de datos (SQL/NoSQL) · Sensores IoT (ultrasónicos, gas, piezo, botones de pánico) · Módulos de IA en computadoras externas (detección de placas, rostros y armas blancas) · Grafana para visualización avanzada.


El resultado es un sistema escalable que pasa de un monitoreo local por cable (Fase 1) a una arquitectura IoT conectada con app móvil (Fase 2) y finalmente a un ecosistema inteligente de ciudad con análisis visual, alertas contextuales, estadísticas históricas y componentes físicos integrados en una maqueta urbana (Fase 3).

Proceso ETL – SG-FOOD (Práctica 1 – SS2)

Proyecto académico enfocado en el proceso ETL (Extract, Transform & Load) basado en datos reales de ventas y compras de la empresa SG-FOOD. El objetivo principal fue transformar datos heterogéneos en información estructurada para análisis empresarial mediante un modelo multidimensional en SQL.

Objetivo principal: integrar diversas fuentes de datos, realizar transformaciones, limpieza y cargar los resultados en un modelo de inteligencia empresarial compatible con análisis multidimensional.

Tecnologías utilizadas: Microsoft SSIS (Visual Studio) · SQL Server · SQL DDL · Scripts de limpieza · Modelo estrella / copo de nieve.

Rol personal: integración de orígenes de datos heterogéneos, diseño del modelo multidimensional en SQL, ejecución de scripts de transformación y validación de carga con consultas analíticas.

Se implementaron consultas clave para analizar ventas, pérdidas por producto, proveedores estratégicos y rendimiento por región. El proyecto culminó con la documentación técnica en GitHub y un README con la descripción del proceso completo.

IMPORCOMGUA – Sistema de Gestión Empresarial (AYD2)

Proyecto académico desarrollado en equipo para el curso Análisis y Diseño de Sistemas 2, aplicando la metodología SCRUM con sprints, daily meetings, retros, planning y backlog estructurado. Se realizó documentación técnica y desarrollo colaborativo bajo roles definidos.

Objetivo del sistema: automatizar los procesos operativos clave de la empresa IMPORCOMGUA (ventas, inventario, pagos, clientes, comisiones) siguiendo principios de arquitectura modular, escalabilidad y eficiencia operativa.

Rol personal: participación en las fases de análisis, documentación (CDU, modelado de base de datos, drivers arquitectónicos y premisas de calidad), codificación de módulos del backend y frontend, además de apoyar en el despliegue con contenedores y CI/CD.

Tecnologías utilizadas:
Docker · GitLab CI/CD · SCRUM · SQL / NoSQL · Git · Arquitectura en capas · Roles y permisos

Se implementó GitLab CI/CD para el despliegue automatizado del proyecto mediante pipelines, lo que permitió integrar integración continua, pruebas e implementación del sistema desde un repositorio centralizado.
